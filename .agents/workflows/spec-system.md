# 🎯 Spec-Driven Development System
You must strictly follow this 4-step workflow for any feature or modification:

1. **PROMPT: /specify** -> Stop coding. Analyze requirements. [cite_start]Create a `spec-[feature].md` file detailing: What, Why, and UI/UX (Shadcn UI + Tailwind). [cite: 11]
2. [cite_start]**PROMPT: /tasks** -> Break down the specification into small, atomic tasks with checkboxes. [cite: 12]
3. **PROMPT: /implement** -> Activate specialized agents to write code. [cite_start]Do not modify legacy code without validation. 
4. **PROMPT: /push** -> The Git Agent must stage all changes, create a professional commit message based on the spec, and push to GitHub.
